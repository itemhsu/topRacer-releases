用 Claude for Education 教學生用 Claude

高中職教師實作指南 · 繁體中文 · 含課前準備清單 · 2026-06

這份指南幫你做到一件事：在課堂上帶學生把 Claude 用得好、用得負責——不是教 AI 理論，而是給你課前準備清單、一堂課的節奏、可直接用的活動與練習。重點放在你最需要的課前準備。

1. 這份指南是給誰的

  • 對象：高中職任何科目的老師（國文、英文、數理、專業群科都適用）。
  • 不需要：你不必先成為 AI 專家——指南把你要做的事拆成清單與步驟。
  • 目標：讓學生學會「把 AI 當思考夥伴，而不是把思考外包給 AI」。

2. 認識 Claude for Education

Claude for Education 是 Anthropic 為教育機構打造的 Claude 版本，核心特色：

功能對課堂的用途
Learning Mode（學習模式）用蘇格拉底式提問引導學生推理，不直接給答案——培養獨立思考，避免「複製貼上」。
Projects（專案）把課程教材、評分規準、範例集中在一個專案，讓對話有一致脈絡。
教師工具產生評分規準（rubric）、給個別化回饋、設計課程與分難度題目。
管理與分析校方層級的存取管理與使用觀察（由學校 IT/行政設定）。
取得方式（誠實說明）：Claude for Education 主要面向大學／機構。高中職若校方尚未導入，實務上可改用 claude.ai（免費或 Pro），一樣能套用本指南的原則——在 prompt 裡明確要求 Claude「用提問引導、先別給答案」，就能手動重現 Learning Mode 的精神。下方所有活動兩種方案都適用。

3. 核心心法：AI Fluency 的「4 D's」

Anthropic 的 AI Fluency 框架用四項能力描述「把 AI 用得好」。把它當成你教學生的骨架——每個活動都對應其中一兩項：

  • Delegation 委派：判斷哪些任務適合交給 AI、哪些該自己做（思考、判斷、創意核心不外包）。
  • Description 描述：把需求、情境、格式講清楚——好的提示 = 一半的成果。
  • Discernment 辨識：批判地評估輸出：正確嗎？有偏誤或幻覺嗎？符合需求嗎？
  • Diligence 盡責：查證、標註來源、為最終結果負責；遵守誠信規範。

4. 🎯 老師課前準備清單

開課前把這份清單跑過一遍（兩種方案都適用）：

A. 存取與政策

  • 確認可用方案：校方是否已導入 Claude for Education；若無，改用 claude.ai（免費／Pro）。
  • 讀過並遵守校方／課程的 AI 使用與資料政策。
  • 確認學生有帳號或裝置；準備無帳號的替代（老師單機螢幕示範、分組共用一台）。

B. 教學設計

  • 開啟 Learning Mode（或在 prompt 要求「引導而非給答案」）。
  • 建立課程 Project，放教材、規準、範例，讓對話有一致脈絡。
  • 寫好今天的學習目標，並對應到一個 4D 能力。
  • 準備 1 個示範 prompt ＋ 1 個「反例」（壞 prompt → 讓學生看差別）。
  • 備好評量 rubric，把「會用 AI」變成可評的指標（見 §8）。

C. 風險與誠信

  • 訂清楚：哪些步驟可用 AI、哪些必須自己做、如何標註。
  • 提醒隱私：不上傳個資、他人作品、未公開或機敏資料。
  • 準備一個「查證幻覺」示範——讓學生親眼看到 AI 也會出錯。

D. 老師自測（最容易被略過、卻最有用）

  • 自己先完整跑一遍學生要做的活動。
  • 估時間、預想學生會卡在哪、先寫好幾個引導問題。

5. 一堂課的節奏範本

6. 課堂活動範例（通用，可套任何科目）

活動 A — 用 Learning Mode 自己解一題 Delegation · Description

目標：體驗「AI 引導我思考」而不是「AI 給我答案」。

  1. 學生挑一題（數學、物理、文法、程式皆可）。
  2. 用下面的 prompt 請 Claude 引導，學生要自己回答每一步。
  3. 結束後寫一句：「我卡在哪、是哪個提問點通了。」
示範 prompt
請用蘇格拉底式提問引導我自己解出這一題,不要直接給答案。每次只問我一個問題,等我回答後再問下一個;如果我答錯,提示我重想,不要替我寫出解法。題目:______

活動 B — 寫作／報告改進（描述–辨識循環） Description · Discernment

目標：練習「修改描述 → 再評估」的反覆循環，學生保留主導權。

  1. 學生貼上自己的一段草稿。
  2. 請 Claude只指出問題並反問，不代寫。
  3. 學生依回饋改一版，再請 Claude 評估——重複 2～3 輪。
示範 prompt
這是我寫的一段:『______』。請先指出其中 2 個最弱的地方,並各問我一個問題,讓我自己改進。先不要幫我重寫,也不要給範例答案。

活動 C — 批判查證一段 AI 輸出 Discernment · Diligence

目標：看見 AI 會犯錯，建立「查證後才採用」的習慣。

  1. 老師（或學生）先讓 Claude 產生一段說明某主題的文字。
  2. 用下面 prompt 找出需查證的宣稱，再請學生用課本／可靠來源實際查證。
  3. 標出哪些正確、哪些有誤或誇大。
示範 prompt
以下是一段對「______」的說明。請列出其中 3 個最需要查證或可能不正確的宣稱,並說明為什麼需要查證。不要直接判斷對錯,讓我自己去查。

7. 學生練習單（可印）

  1. 委派判斷：列出今天作業的 5 個步驟，標出哪些「可用 AI 協助」、哪些「必須自己完成」，並說明理由。
  2. 描述升級：把一個爛 prompt（例：「幫我寫報告」）改寫成好的 prompt（含主題、對象、長度、格式、限制），比較兩次輸出差別。
  3. 抓幻覺：找出 Claude 一段回答中 1 個你查證後發現有問題的地方，附上你的查證來源。

8. 學術誠信與負責任使用

  • 標註：規定學生註明「哪裡用了 AI、怎麼用」；把它當成像查字典一樣的公開工具，而非隱瞞的捷徑。
  • 查證：AI 會「一本正經地說錯話」（幻覺）——採用前一定查證。
  • 隱私：不上傳個資、他人作品、未公開或機敏資料。
  • 不外包思考：核心判斷、創意、論證要由學生自己完成；AI 是夥伴不是代工。
  • 遵守規範：以校方與該科課程的規定為準，課前講清楚界線。
