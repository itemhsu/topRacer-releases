切點行為獎勵 · Tangent-Point Steering Reward

topRacer 獎勵函數設計規格書 · 2026-06-28 · 通用版（不綁特定賽道）

動態視線切點自動取彎內側 二元獎勵 0 / 1容忍 ±6° / ±3° 純轉向方向項邊界由 waypoints 推導

一句話

獎勵的依據是轉向指令是否為「把車頭轉回切點」的正確修正動作，而不是車頭當下對不對準切點。 對準了卻還在轉 → 不給；沒對準但往切點方向轉 → 給。 切點是「從車子位置對下一個彎內側邊界做的視線支撐切線」的觸點，會隨車前進而沿邊界滑動（真實賽車線）。

1. 目標與動機

讓 policy 學會「把車頭對著下一個彎的切點開」這個轉向控制法則。一般的中線獎勵只獎勵「貼著中線」， 學不出賽車線（晚 apex、切內側）。本獎勵把正確的轉向修正動作直接變成訊號：偏了就獎勵轉回、對準就獎勵維持， 而且只看轉向指令，不混入速度或貼線，讓「轉向方向」這件事被獨立、乾淨地塑形。

2. 賽道與座標：邊界由 waypoints 推導（通用）

DeepRacer 的 reward_function(params) 只會收到中線 params["waypoints"] 與平均道寬 params["track_width"]，拿不到內/外邊界陣列。因此本設計不烤任何特定賽道座標， 而是每條賽道都從中線即時推導邊界 —— 任何 DeepRacer 賽道皆可直接套用。

2.1 推導公式

對第 i 個中線點，取局部行進方向的左法線 nL，沿法線偏移半個道寬即得兩側邊界：

t   = center[i+1] - center[i-1]            # 局部行進方向（環狀索引）
nL  = normalize( (-t.y,  t.x) )            # 左法線（單位向量）
half = track_width / 2

inner[i] = center[i] + half * nL           # 左側邊界
outer[i] = center[i] - half * nL           # 右側邊界
近似說明：track_width 是平均道寬，真實寬度沿賽道有些微變化，故推導邊界與實際邊界會有公分級偏差。 對「±6° 轉向方向」這種容忍度可忽略。若日後要高擬真，可選擇性改讀真實 npy 邊界，但預設走通用版。

3. 第一次呼叫的快取：四張表

比照現有 Apex Brake Speed 的做法，第一次呼叫時用中線算一次並快取（之後每步直接查）：

快取內容怎麼算
_LEFT左彎索引集合有號曲率 turn3(wp[i-2],wp[i],wp[i+2]) > eps
_RIGHT右彎索引集合turn3(...) < -eps（eps = 0.15）
_ENTRIES彎道入口表：每個彎的起點索引本身是彎、前一點不是彎：i∈彎 且 (i-1)∉彎
_APEX每個彎索引預先算好的 apex 內側點 (x,y)左彎 center+½w·nL、右彎 center−½w·nL

中線是靜態的，四張表算一次即可。實測 2018 共偵測到 5 個彎（見圖 1）。

4. 動態視線切點（每步）

4.1 找「下一個彎」

由車子的 closest_waypoints = [prev, next] 取 next，在 _ENTRIES 中找車前方最近、尚未通過的彎入口（環狀；若已在彎中則取當前彎）。 取該彎所有索引對應的 _APEX 點，組成 apex-side 邊界折線 B = [B₀ … Bₘ]。

4.2 支撐切線求切點 T

切點 T = 邊界折線上「整段邊界都落在視線 P→T 同一側」的那個頂點（凸弧的支撐觸點）。用外積判定，O(m) 掃描： 

def cross(p, a, b):                  # (a-p) × (b-p)
    return (a.x-p.x)*(b.y-p.y) - (a.y-p.y)*(b.x-p.x)

# 左彎：取「所有其他邊界點都在視線左側」的支撐點（最順時針極端）
# 右彎：取「所有其他邊界點都在視線右側」的支撐點（最逆時針極端）
def tangent_point(P, B, left_turn):
    best = B[0]
    for cand in B:
        ok = all( (cross(P, cand, other) >= 0) == left_turn          # 同側測試
                  for other in B if other is not cand )
        if ok:
            best = cand; break
    return best

這個支撐切點就是駕駛學的 limit point。關鍵：車要靠外側進彎 —— 從外側看出去，支撐切點正好落在 apex，且切線只在 apex 輕觸內側邊界（相切非穿越），車沿切線過彎不會撞牆。直線路段時下一彎很遠 → 切點幾乎在正前方 → e≈0 → 直行（車自然保持靠外側）。隨車過彎，limit point 沿邊界滑向出口。

4.3 對準誤差 e

bearing = degrees( atan2(T.y - P.y, T.x - P.x) )
e = normalize_deg( bearing - heading )       # (-180, 180]，正=切點在車頭左方

5. 必要動作與獎勵（核心）

5.1 由 e 決定「必要動作」

容忍值：對準死區 TOL_ALIGN = 6°，直行帶 TOL_STRAIGHT = 3°。

對準狀態條件必要動作
車頭在切點右方e > +6°左轉
車頭在切點左方e < −6°右轉
已對準|e| ≤ 6°直行

5.2 二元獎勵（依據是轉向指令，不是對準狀態）

讀 steering = params["steering_angle"]（正＝左、負＝右）。方向對 → 1.0，否則 → 0.0：

必要動作給 1.0（方向對）給 0.0（沒做對）
左轉steering > +3°直行帶內 或 右轉
右轉steering < −3°直行帶內 或 左轉
直行|steering| ≤ 3°任何 |steering| > 3°
語意檢查：對準卻轉彎 → 0（直行列，|steering|>3）；沒對準但往切點方向轉 → 1.0（左/右轉列）。 完全符合「轉向角度才是給獎勵的要素」。

6. 虛擬碼（完整流程）

_CACHE = None   # (_LEFT, _RIGHT, _ENTRIES, _APEX, corner_dir)

def reward_function(params):
    global _CACHE
    wp   = params["waypoints"]
    w    = params["track_width"]
    if _CACHE is None:
        _CACHE = build_cache(wp, w)          # 四張表，只算一次
    _LEFT, _RIGHT, _ENTRIES, _APEX, _DIR = _CACHE

    P       = (params["x"], params["y"])
    heading = params["heading"]
    steer   = params["steering_angle"]
    _, nxt  = params["closest_waypoints"]

    corner  = next_corner(nxt, _ENTRIES, _DIR)     # 車前方最近的彎
    B       = apex_polyline(corner, _APEX)         # apex-side 邊界折線
    T       = tangent_point(P, B, _DIR[corner]=='L')
    e       = normalize_deg( bearing(P, T) - heading )

    if   e >  TOL_ALIGN:   return 1.0 if steer >  TOL_STRAIGHT else 0.0   # 應左轉
    elif e < -TOL_ALIGN:   return 1.0 if steer < -TOL_STRAIGHT else 0.0   # 應右轉
    else:                  return 1.0 if abs(steer) <= TOL_STRAIGHT else 0.0  # 應直行

TOL_ALIGN, TOL_STRAIGHT = 6.0, 3.0

7. 邊界情況

情況處置
整圈幾乎無彎 / _ENTRIES 為空找不到彎 → 視為直行需求：|steering| ≤ 3° 給 1.0。
車正位於彎中next_corner 回傳當前所在彎；T 仍由支撐切線求得（會落在前段邊界）。
支撐測試找不到唯一點（共線/雜訊）退化為取 apex 折線中點，保證有 T、不丟例外。
倒著開 is_reversedenv 已在倒車時終止 episode；本獎勵照常計算（不另外處理）。
離道 is_offtrack純轉向項不含 on-track；離道由 env 的 off-track 終止自然懲罰。
純轉向項的已知取捨（A 案，使用者拍板）：本獎勵不含前進或貼線誘因。env 在「全輪離道」會終止 episode（離道有自然懲罰）， 但沒有前進獎勵 → 理論上車可能慢速或原地微調騙取直行/修正獎勵。建議實戰時把本獎勵與速度／進度獎勵相加， 本項專責「轉向方向」的塑形。
